# export-md 命令使用说明

## 功能概述

`export-md` 命令支持两种导出模式：
1. **按数量导出** - 导出指定数量的最新帖子
2. **按日期导出** - 导出指定日期的所有帖子

**新特性**：
- 每80个帖子（4页）自动生成一个新的MD文件
- 附件按日期分目录存储
- 支持最多3次重试机制（分页请求和下载请求）

## 命令格式

```bash
node fetch_topics.js export-md <group_id> <count|YYYY-MM-DD> [scope] [output_dir]
```

## 参数说明

- `group_id` (必需) - 星球 ID
- `count|YYYY-MM-DD` (必需) - 数量或日期
  - 数量：如 `30` 表示导出最新 30 条帖子
  - 日期：如 `2026-03-01` 表示导出该日期的所有帖子
- `scope` (可选) - 帖子范围，默认 `all`
  - `all` - 所有帖子
  - `digests` - 仅精华帖
- `output_dir` (可选) - 输出目录，默认 `archive`

## 文件命名规则

### 按数量导出
- 文件名格式：`MM-DD-HH-mm-ss-0x.md`
- 示例：
  - `03-01-14-30-45-01.md` (第1个文件，1-80个帖子)
  - `03-01-14-30-45-02.md` (第2个文件，81-160个帖子)
  - `03-01-14-30-45-03.md` (第3个文件，161-240个帖子)

### 按日期导出
- 文件名格式：`MM-DD-0x.md`
- 示例：
  - `03-01-01.md` (第1个文件，1-80个帖子)
  - `03-01-02.md` (第2个文件，81-160个帖子)
  - `03-01-03.md` (第3个文件，161-240个帖子)

## 目录结构

```
archive/
├── 03-01-01.md                                # 按日期导出的第1个文件
├── 03-01-02.md                                # 按日期导出的第2个文件
├── 03-01-14-30-45-01.md                       # 按数量导出的第1个文件
└── asset/                                      # 附件根目录
    └── 03-01/                                  # 按日期分目录存储
        ├── 82811515254842180_01_image_xxx.png # topic_id_序号_类型_id.扩展名
        ├── 82811515254842180_02_file_xxx.pdf
        ├── 22811515254842420_01_image_yyy.jpg
        └── ...
```

## 分文件规则

- **每80个帖子生成一个新文件**（每页20个帖子，4页 = 80个帖子）
- 文件序号从 `01` 开始递增
- 每个文件独立包含完整的头部信息
- 所有文件的附件存储在同一个日期目录下

## 使用示例

### 1. 按数量导出（导出最新 30 条帖子）

```bash
node fetch_topics.js export-md 51122188845424 30 all
```

输出文件：`archive/03-01-14-30-45-01.md`（只有1个文件，因为不足80个帖子）

### 2. 按数量导出（导出最新 150 条帖子）

```bash
node fetch_topics.js export-md 51122188845424 150 all
```

输出文件：
- `archive/03-01-14-30-45-01.md`（1-80个帖子）
- `archive/03-01-14-30-45-02.md`（81-150个帖子）

### 3. 按日期导出（导出 2026-03-01 的所有帖子）

```bash
node fetch_topics.js export-md 51122188845424 2026-03-01 all
```

输出文件：
- `archive/03-01-01.md`（1-80个帖子）
- `archive/03-01-02.md`（81-160个帖子）
- ...（根据实际帖子数量）

### 4. 自定义输出目录

```bash
node fetch_topics.js export-md 51122188845424 30 all ./my-export
```

输出文件：`my-export/03-01-14-30-45-01.md`

### 5. 导出精华帖

```bash
node fetch_topics.js export-md 51122188845424 2026-03-01 digests
```

## 重试机制

### 分页请求重试
- **触发条件**：API 返回 `!data.succeeded` 错误
- **重试次数**：最多3次
- **重试间隔**：2秒
- **重试策略**：使用相同的URL重试，成功后重置计数器

### 下载请求重试
- **触发条件**：`getFileDownloadUrl` 或 `downloadBinaryFromUrl` 失败
- **重试次数**：最多3次
- **重试间隔**：指数退避（2秒、4秒、8秒）
- **重试策略**：每次失败后等待更长时间再重试

## 延迟策略

1. **分页请求之间**：随机停顿 2-5 秒
2. **下载请求之间**：固定停顿 1 秒
3. **重试等待**：
   - 分页重试：固定 2 秒
   - 下载重试：指数退避（2秒、4秒、8秒）

## 注意事项

1. **日期格式**：必须严格使用 `YYYY-MM-DD` 格式（如 `2026-03-01`）
2. **时区**：使用帖子的 `create_time` 字段进行日期匹配，注意时区差异
3. **分页机制**：每次获取20条帖子，使用 `end_time` 参数进行分页
4. **附件存储**：
   - 按日期分目录存储在 `asset/MM-DD/` 目录
   - 文件名格式为 `{topic_id}_{序号}_{类型}_{id}.{扩展名}`
   - 同一天的所有附件存储在同一个目录下
5. **流式处理**：边获取边下载边写入，防止 URL 过期，降低内存占用
6. **失败处理**：
   - 部分附件下载失败不会中断整体流程
   - 失败信息会记录在输出的 JSON 中
   - 分页请求失败会自动重试最多3次
7. **文件分割**：
   - 每80个帖子自动创建新文件
   - 文件序号自动递增
   - 每个文件包含独立的头部信息

## 环境变量

确保已设置 `ZSXQ_TOKEN` 环境变量：

```bash
export ZSXQ_TOKEN="你的token值"
```

或在 `~/.openclaw/.env` 中配置：

```
ZSXQ_TOKEN=你的token值
```
